Pulsar Debezium同步MySQL数据

Pulsar本地运行

参考《单节点安装运行Pulsar.md》

Mysql的配置

  • 单机版
    • 单机Mysql必须启用binlog
    • A Debezium MySQL connector requires a MySQL user account. This MySQL user must have appropriate permissions on all databases for which the Debezium MySQL connector captures changes.
  • 主从
    • 仅读取一个示例。如果该服务器发生故障，则必须重新启动或恢复该服务器，然后连接器才能继续。
  • 集群
    • MySQL有多种高可用性解决方案，它们使容忍性大大提高，几乎可以从问题和故障中立即恢复。大多数HA MySQL群集使用GTID，以便副本能够跟踪任何主服务器上的所有更改。
#创建一个mysql用户角色
mysql> CREATE USER 'user'@'localhost' IDENTIFIED BY 'password';

#赋予角色权限reload 和 flush_tables 
mysql> GRANT SELECT, RELOAD,flush_tables, SHOW DATABASES, REPLICATION SLAVE, REPLICATION CLIENT ON *.* TO 'user' IDENTIFIED BY 'password';

#刷新权限
mysql> FLUSH PRIVILEGES;

#启用binlog
#检查该log-bin选项是否已启用：
mysql> SELECT variable_value as "BINARY LOGGING STATUS (log-bin) ::"
FROM information_schema.global_variables WHERE variable_name='log_bin';

#如果是OFF，请使用以下属性配置MySQL服务器配置文件
server-id = 223344 #对于server-idMySQL群集中的每个服务器和复制客户端，该值必须唯一。在设置MySQL连接器期间，Debezium会为连接器分配一个唯一的服务器ID。
log_bin    = mysql-bin #log_bin是二进制日志文件序列的基本名称
binlog_format     = ROW #在binlog-format必须设置为ROW或row。
binlog_row_image  = FULL #在binlog_row_image必须设置为FULL或full。
expire_logs_days  = 10 #这是自动删除binlog文件的天数。默认值为0，表示不自动删除。

#再次检查binlog状态，以确认更改：
mysql> SELECT variable_value as "BINARY LOGGING STATUS (log-bin) ::"
FROM information_schema.global_variables WHERE variable_name='log_bin';

Debezium相关下载与配置

首次创建步骤：

  1. 需要保证pulsar运行正常（zookeeper、bookie、broker、functions-worker）
  2. 根据functions_worker.conf的connectorsDirectory配置上传connector的nar包，一般在pulsar根目录下的connectors目录。官方下载地址
  3. 配置yml文件，一般放在conf/functions目录下
  4. 创建source
bin/pulsar-admin source create --source-config-file conf/functions/cdc.yml
  1. 查看对应topic是否创建完成
bin/pulsar-admin topics list tenant/namespace

yml文件示例

## 租户
tenant: "tenant"
## 命名空间
namespace: "namespace"
## source 名称
name: "source-cdc-demo"
## source topics名
topicName: "tenant/namespace/cdc-mysql"
## nar包连接器地址
archive: "connectors/pulsar-io-debezium-mysql-2.7.1.nar" 
## 并行
parallelism: 1 

configs:
  ## 数据库相关基础配置
  database.hostname: "host"
  database.port: "3306" 
  database.user: "username"
  database.password: "password"
  ## 对应上面的server-id
  database.server.id: "654321"
  ## 服务名，会拼接topic
  database.server.name: "durian"
  ## 包含的数据库
  database.include.list: "gscm"
  ## 包含的数据库表（注意要带上库名因为可以包含多个库）
  table.include.list: "gscm.sys_user" 
  ## 使用历史处理类
  database.history: "org.apache.pulsar.io.debezium.PulsarDatabaseHistory"
  ## 历史topic
  database.history.pulsar.topic: "tenant/namespace/__history"
  ## 管理历史topic的broker地址
  database.history.pulsar.service.url: "pulsar://host:6650"
  ## ssl连接模式
  database.ssl.mode: preferred

  ## KEY_CONVERTER_CLASS_CONFIG, VALUE_CONVERTER_CLASS_CONFIG
  key.converter: "org.apache.kafka.connect.json.JsonConverter"
  value.converter: "org.apache.kafka.connect.json.JsonConverter"
  ## Pulsar broker地址
  pulsar.service.url: "pulsar://host:6650"
  ## 偏移topic
  offset.storage.topic: "tenant/namespace/__offset"
  ## 配置decimal取值
  decimal.handling.mode: "string"
  ## 配置快照模式
  snapshot.mode: when_needed
