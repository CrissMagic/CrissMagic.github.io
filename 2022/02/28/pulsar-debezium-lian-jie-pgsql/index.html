Pulsar Debezium同步PgSQL数据

好久不见，目前正在打造公司这边的数据同步方案，在CDC这边积累了一些经验，后续几篇将做个介绍。本文主要介绍如何使用Debezium连接到PG数据库。
当前Debezium使用分为以下几种方式：

  1. Pulsar IO Source Connector - 使用Pulsar IO的Source进行连接，Pulsar对Debezium的包做了封装，是本文主要介绍的方式。
  2. Kafka Connector - 同1，只不过MQ换为了Kafka，相对于Pulsar，Kafka在对接方面会更适配，Debezium最初的开发也是对接的Kafka。
  3. Debezium Server - 直接在服务器安装Debezium的服务，Sink可根据配置发送到不同的MQ或者NO SQL
  4. Debezium API - 使用Debezium提供的API手动创建复制槽，缺点是需要手动控制的东西比较多

安装Pulsar（本地或者集群）

此步骤参考对应的文档即可，注意，PG本身需要修改复制等级、host等配置，此部分参考官方文档即可。

Debezium相关下载与配置

主要关键文件有3个

  1. Debezium PostgreSQL CDC source
    官方下载地址
    下载的nar包统一放在${pulsar安装目录}/connectors下
  2. Source 配置
    目前习惯放在${pulsar安装目录}/conf/functions/${当前环境，如dev、test、prod}/cdc/source目录下。目前没有使用sink，所以会少一级目录
    使用yaml格式配置，Debezium所需要的配置
    官方配置
  3. 启动脚本。主要是便于管理Source
    目前习惯放在~/workspace/pulsar/cdc/${当前环境}目录下，主要是对Source的创建、删除、更新、状态查看、暂停、启动
    目前基本统一，只需要根据实际情况更改脚本前面的变量
    完成配置后可以直接sh 脚本.sh create启动，注意，status查看时会打印出该Source的日志服务器host，可在对应服务器上的${pulsar安装目录}/logs/functions/目录查看日志信息

Pulsar Source配置

以下为配置的介绍

## 租户信息
tenant: "tenant"
## 命名空间
namespace: "namespace"
## source name
name: "source-new"
## 整体信息的Topic
topicName: "public/default/cdc-topic"
## nar包位置
archive: "connectors/pulsar-io-debezium-postgres-2.7.3.nar"
## 并发数
parallelism: 1
## java运行参数，该配置限制Source内存使用
runtimeFlags: "-Xms128M -Xmx256M -XX:MaxDirectMemorySize=256m"

configs:
    ## 数据库连接配置
    database.hostname: "hostname"
    database.port: "5432"
    database.user: "username"
    database.password: "password"
    database.server.name: "server_name"
    database.dbname: "dbname"
    ## 设置发布名称，建议根据实际情况配置，并添加cdc相关信息，方便监控时看到使用的程序
    publication.name: "cdc_dbname" 
    ## 配置复制槽名称，建议根据实际情况配置，并添加cdc相关信息，方便监控时看到使用的程序
    slot.name: "cdc_dbname"

    ## 设置需要同步的schema
    schema.include.list: "db_schema"
    ## 设置需要同步的表，,分隔，必须包含schema信息
    table.include.list: "db_schema.table1,db_schema.table2"

    ## Pulsar Config
    pulsar.service.url: "pulsar://pulsarUrl:6650"
    ## History相关设置
    database.history: "org.apache.pulsar.io.debezium.PulsarDatabaseHistory"
    database.history.pulsar.topic: "public/default/__dbname-history"
    database.history.pulsar.service.url: "pulsar://pulsarUrl:6650"
    ## offset设置，注意，一般情况下history、offset、topic这三个Topic不要删除，否则会导致偏移量丢失
    offset.storage.topic: "public/default/__dbname-offset"

    ## 其他debezium配置
    ## 使用pgoutput插件
    plugin.name: "pgoutput"
    ## 快照生成策略，默认为`initial`，看情况选择`always`或者`initial_only`，具体描述参考官方文档
    snapshot.mode: "always"
    ## 监听间隔，默认1秒
    poll.interval.ms: 1000
    ## 心跳监测间隔，默认关闭，目前设置为30秒
    heartbeat.interval.ms: 30000
    ## 心跳表插入，用于防止表数据不发生变化时WAL过大，当前场景不需要
    #heartbeat.action.query: 'INSERT INTO "public"."debezium_heartbeat"(time) values(extract(epoch from current_timestamp)::int);'
    ## 控制decimal取值
    decimal.handling.mode: "string"
    ## 设置时间格式
    time.democision.mode: "connect"

管理脚本

以下为管理脚本的示例，可根据自身情况修改，注意要和上面的配置文件对应！

#!/bin/bash
TENANTS=tenant
NAMESPACE=namespace
SOURCE_NAME=source-name
CONFIG_FILENAME=demo/cdc-pgsql-config.yml

PULSAR_PATH=/usr/local/apache-pulsar

status() {
  cd $PULSAR_PATH
  echo "查看CDC Connector: "$SOURCE_NAME
  ./bin/pulsar-admin source status --name $SOURCE_NAME --tenant $TENANTS --namespace $NAMESPACE
}

stop() {
  cd $PULSAR_PATH
  echo "停止CDC Connector: "$SOURCE_NAME
  ./bin/pulsar-admin source stop --name $SOURCE_NAME --tenant $TENANTS --namespace $NAMESPACE
}

start() {
    cd $PULSAR_PATH
  echo "开始CDC Connector: "$SOURCE_NAME
  ./bin/pulsar-admin source start --name $SOURCE_NAME --tenant $TENANTS --namespace $NAMESPACE
}

restart() {
    cd $PULSAR_PATH
  echo "重启CDC Connector: "$SOURCE_NAME
  ./bin/pulsar-admin source restart --name $SOURCE_NAME --tenant $TENANTS --namespace $NAMESPACE
}

create() {
    cd $PULSAR_PATH
  echo "创建CDC Connector: "$SOURCE_NAME
  ./bin/pulsar-admin source create --source-config-file ./conf/functions/$CONFIG_FILENAME
  echo "当前topic如下"
  ./bin/pulsar-admin topics list $TENANTS/$NAMESPACE
}

update() {
    cd $PULSAR_PATH
  echo "更新CDC Connector: "$SOURCE_NAME
  ./bin/pulsar-admin source update --name $SOURCE_NAME --tenant $TENANTS --namespace $NAMESPACE --source-config-file ./conf/functions/$CONFIG_FILENAME
  echo "当前topic如下"
  ./bin/pulsar-admin topics list $TENANTS/$NAMESPACE
}

delete() {
    cd $PULSAR_PATH
  echo "删除CDC Connector: "$SOURCE_NAME
  ./bin/pulsar-admin source delete --name $SOURCE_NAME --tenant $TENANTS --namespace $NAMESPACE
}

case $1 in
status)
        status
;;
stop)
        stop
;;
start)
        start
;;
restart)
        restart
;;
create)
        create
;;
update)
        update
;;
delete)
        delete
;;
esac
