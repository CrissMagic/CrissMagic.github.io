Vue 标签拖拽实现与部分踩坑记录

前言

自己作为后端工程师开发前端项目多有不易，幸好目前前端提供的框架还算多，抛开一些类似“圣诞事件”的不提，在使用 vue+elementui 做了几个项目后还是感觉不错，不过 vue 的重渲染有点麻烦，关于 elementui 的讨论也有很多，自己最近也试着接触了例如 vuetify、quasar、taro，感觉还是更偏向于移动端的 ui 开发，不过在下次项目选型的时候会更加注意，特别是政府项目。
七月底了，抽时间记录一下在最近在开发过程中一些坑还有实现标签拖拽的代码，供以后参考。

标签拖拽实现

目的是用户可以自行从页面左侧选择需要的标签拖拽到右侧的方框里，使用 vue 绑定配置 echarts 的 option。

效果如下图所示：

本身想使用 vuedragable 插件的，但是发现或许不适合在多个容器间进行拖拽，所以此处采用 dargula，但由于该技术并未提供拖拽后的回调函数，所以采用手动添加监听事件的方式。

具体功能逻辑如下：

  1. 新增系列时向 v-for 绑定的数组中添加对象，对象中设置该系列对应的容器 id；
  2. 在下个周期渲染完成后根据 class 类名获取 dom 元素并添加至 dragula 容器；
  3. 同样手动给这些 dom 添加 DOMNodeInserted 事件和 DOMNodeRemoved 事件，并用参数传递标签 id 和容器 id；
  4. 在监听事件触发后执行对应的逻辑方法。

具体代码如下（已省略不重要的代码）：


export default {
	...
	created() {
	// 初始化dragula
	this.drag = this.$dragula(
	{
  		isContainer: function (el) {
		// 设置带有tagData的class样式的dom加入容器
	        return el.classList.contains('tag-data');
        },
        	moves: function (el, source, handle, sibling) {
	        // 哪些元素能拖动，true为都能拖动
	        return true;
        },
        	accepts: function (el, target, source, sibling) {
          	// 放入容器的条件，true为都能放入
          	return true;
        },
        	invalid: function (el, handle) {
          	// 默认情况下不会阻止任何拖动
          	return false;
        },
        // 拖动的元素是否为副本
        copy: false,
        // 复制的源容器重新排序
        copySortSource: false,
        // 是否将拖到容器外的元素放回原处
        revertOnSpill: true,
        // 是否将拖到容器外的元素删除
        removeOnSpill: false,
        // 设置获取附加镜像元素的元素
        mirrorContainer: document.body
      })
	},
	methods:{
	...mapActions(
        // 映射actions中的方法
        {
          addContainerTagArray: 'addContainerTagArray',
          removeContainerTagArray: 'removeContainerTagArray'
        }
      ),
	addAndBind() {
        // 先清空之前的数据
        this.seriesArray = []
        for (let i = 0; i < this.currentSeriesNum; i++) {
        // 随机生成UUID
          let tempUUID = genUUID()
          this.seriesArray.push(
            {
              containerId: tempUUID,
              type: '',
              lineStyle: {
                color: '#409EFF'
              }
            }
          )
        }
        let _this = this
        // 此处必须使用$nextTick否则vue还未渲染完时无法通过getElementsByClassName方法获取到dom对象
        this.$nextTick(() => {
          let eles = document.getElementsByClassName('series-tag-container');
          for (let i = 0; i < _this.seriesArray.length; i++) {
            // 数据添加完统一添加事件监听
            let series = _this.seriesArray[i]
            // dom渲染完成后执行
            let element = eles[i]
            // 添加拖拽容器
            _this.drag.containers.push(element)
            // 设置自定义属性
            element.setAttribute("containerId", series.containerId)
            //  给新的系列标签容器添加监听器
            element.addEventListener("DOMNodeInserted", function (e) {
              if (element.children.length > 1) {
                // 已经存在一个标签则退回
                _this.drag.cancel()
                _this.$message({
                  message: '最多选择一个标签',
                  type: 'warning'
                });
                return
              }
              _this.addSeriesArray(e.target.getAttribute("id"),e.currentTarget.getAttribute("containerId"))
            })
            element.addEventListener("DOMNodeRemoved", function (e) {
              _this.removeSeriesArray(e.currentTarget.getAttribute("containerId"))
            })
          }
        })
      },
      addSeriesArray(tagId, containerId) {
        // 从tagArray中根据id添加数据
        let tempTag = this.tagArray.filter(tag => tag.id == tagId)
        for (let i = 0; i < this.seriesArray.length; i++) {
          let series = this.seriesArray[i]
          if (series.containerId == containerId) {
          // 容器id相等则设置标签数据
            series.id = tempTag[0].id
            series.name = tempTag[0].fieldName
            series.data = tempTag[0].data
            series.label = {
              show: true,
              position: 'top'
            }
          }
        }
        // 将容器ID与标签ID添加到store，可以用vuex来管理标签id与容器id的关系，但是此处未用到
        // this.addContainerTagArray({
        //   containerId: containerId,
        //   tagId: tagId
        // })
      },
      removeSeriesArray(containerId) {
      // 标签移出时
        for (let i = 0; i < this.seriesArray.length; i++) {
          let temp = this.seriesArray[i]
          if (temp.containerId == containerId) {
            // 匹配则初始化
            temp.id = ''
            temp.data = []
            temp.name = ''
          }
        }
        // 从store中移出
        // this.removeContainerTagArray(containerId)
      }
   }
}

麻烦之处在于：

  1. Vue 的重渲染，如果动态添加系列数量，那么在系列数组发生变化后会导致整体的父组件重新渲染，导致拖拽来的标签消失（但实际数据还在），所以此处只能在一开始指定系列数量，并且如果要改变会提醒会丢失已有的配置；
  2. dragula 没有提供时间毁掉所以只能手动添加监听事件，需要注意的是由于添加进系列数组和绑定监听事件均是在同一方法内，此时 vue 还未对 dom 进行重渲染，所以此时是无法获取到 dom 对象的，需要使用 $nextTick 在渲染完成后才能获取，当然别忘记 this 要换成 _this

一些坑

popover 弹框组件在循环中的使用

elementUI 官方提供的 popover 的事例只是在单一按钮的基础上绑定弹框，但是更多的情况是在表格或者 v-for 中使用，那么此时就不能按照官方的写法来。完整代码如下：

<el-table
              :data="chartArray"
              :highlight-current-row="true"
              @row-click="setOption"
              style="width: 100%">
              <el-table-column
                label="图表名"
                width="110">
                <template slot-scope="scope">
                  {{scope.row.name}}
                </template>
              </el-table-column>
              <el-table-column label="操作">
                <template slot-scope="scope">
                  <el-button
                    size="mini"
                    type="primary"
                    @click="handleEdit(scope.$index, scope.row)">编辑
                  </el-button>
                  <el-popover placement="top" width="160" :ref="'popover-'+scope.$index">
                    <p>确定删除吗？</p>
                    <div style="text-align: right; margin: 0">
                      <el-button
                        size="mini"
                        type="text"
                        @click="scope._self.$refs['popover-'+scope.$index].doClose()">
                        取消
                      </el-button>
                      <el-button
                        type="primary"
                        size="mini"
                        @click="scope._self.$refs['popover-'+scope.$index].doClose();;handleDelete(scope.row)">
                        确定
                      </el-button>
                    </div>
                    <el-button type="danger" size="mini" slot="reference">删除</el-button>
                  </el-popover>
                </template>
              </el-table-column>
            </el-table>

关键之处在于：

  1. <el-popover> 组件上的 ref 属性在循环的过程中需要绑定不同的值，可以用循环的 index 或者使用循环对象的 id；
  2. 在点击取消或者确定时使用 scope._self.$refs['popover-'+scope.$index].doClose() 来关闭对应的 popover，后面再加上本身要执行的方法。

Echarts 的 option 合并

echarts 在默认情况下如果切换 option 会将之前的 option 与现在的 option 进行合并，因为一般情况下都是系列 series 发生变化，其他的如 xy 轴，图例之类的不会变，如果此时进行不同的 option 切换就会出现以下的结果：

官方文档中是这么说的：option 合并
所以如果不想进行合并需要设置一下：

setOption(row, column, event) {
        if (!this.chart) {
          this.chart = this.$echarts.init(document.getElementById('chart-preview-container'))
        }
        this.chart.setOption(row.option, {
          // option更新时不合并
          notMerge: true
        })
      },
