CentOS7下Docker相关操作

写在前面

  • Docker目前使用越来越多，并且渐渐熟悉后也体会到了一些方便快捷之处，此处记录一下目前接触到的相关操作。

    安装Docker

  1. 查看内核，版本必须高于3.10
    uname -r
  2. 移除旧的版本
    sudo yum remove docker*
  3. 安装一些必要的系统工具
    sudo yum install -y yum-utils device-mapper-persistent-data lvm2
  4. 添加软件源信息（这里拉取阿里云的镜像）
    sudo yum-config-manager --add-repo http://mirrors.aliyun.com/docker-ce/linux/centos/docker-ce.repo
  5. 更新 yum 缓存
    sudo yum makecache fast
  6. 安装 Docker-ce（社区版）
    sudo yum -y install docker-ce
  7. 启动 Docker 后台服务
    sudo systemctl start docker
  8. 测试运行 hello-world
    docker run hello-world
  9. 添加开机启动
    sudo systemctl enable docker

    简易安装mysql5.7

  10. 拉取官方镜像
    docker pull mysql:5.7
  11. 创建映射目录
    # 创建映射目录
mkdir -p /root/mysql/data /root/mysql/logs /root/mysql/conf
# 创建配置文件
cd /root/mysql/conf
touch my.cnf
  12. 创建并运行容器（注意设置密码）
    docker run -p 3306:3306 --name mysql -v /root/mysql/conf:/etc/mysql/conf.d -v /root/mysql/logs:/logs -v /root/mysql/data:/var/lib/mysql -e MYSQL_ROOT_PASSWORD=root -d mysql:5.7
  13. 远程连接

SpringBoot项目打包Docker镜像

此处参考该开源项目

手动打包

手动打包需要确保打包环境下有安装Docker

  1. 制作Dockerfile，例子如下：
# 基础镜像
FROM openjdk:8-jdk-alpine

# 作者信息
MAINTAINER "Yangkai.Shen 237497819@qq.com"

# 添加一个存储空间
VOLUME /tmp

# 暴露8080端口
EXPOSE 8080

# 添加变量，如果使用dockerfile-maven-plugin，则会自动替换这里的变量内容
ARG JAR_FILE=target/spring-boot-demo-docker.jar

# 往容器中添加jar包
ADD ${JAR_FILE} app.jar

# 启动镜像自动运行程序
ENTRYPOINT ["java","-Djava.security.egd=file:/dev/urandom","-jar","/app.jar"]
  1. 打包
docker build -t spring-boot-demo-docker .
  1. 运行
docker run -d -p 9090:8080 spring-boot-demo-docker

使用maven插件打包

使用maven插件打包须确保打包环境下有安装docker和maven

  1. pom.xml中添加插件
<properties>
    <dockerfile-version>1.4.9</dockerfile-version>
</properties>

<plugins>      
    <plugin>
        <groupId>com.spotify</groupId>
        <artifactId>dockerfile-maven-plugin</artifactId>
        <version>dockerfile-version</version>
        <configuration>
            <repository>project.build.finalName</repository>
            <tag>project.version</tag>
            <buildArgs>
                <JAR_FILE>target/project.build.finalName.jar</JAR_FILE>
            </buildArgs>
        </configuration>
        <executions>
            <execution>
                <id>default</id>
                <phase>package</phase>
                <goals>
                    <goal>build</goal>
                </goals>
            </execution>
        </executions>
    </plugin>
</plugins>
  1. 执行mvn打包命令，因为插件中 execution 节点配置了 package，所以会在打包的时候自动执行 build 命令
mvn clean package -Dmaven.test.skip=true
  1. 查看镜像
docker images
  1. 运行
docker run -d -p 9090:8080 spring-boot-demo-docker:1.0.0-SNAPSHOT

本地单机搭建Portainer

Portainer介绍

Portainer是Docker的图形化管理工具，提供状态显示面板、应用模板快速部署、容器镜像网络数据卷的基本操作（包括上传下载镜像，创建容器等操作）、事件日志显示、容器控制台操作、Swarm集群和服务等集中管理和操作、登录用户管理和控制等功能。功能十分全面，基本能满足中小型单位对容器管理的全部需求。

单机版运行

  1. pull镜像
docker pull docker.io/portainer/portainer
  1. 运行容器
docker run -d -p 9000:9000 \
--restart=always \
-v /var/run/docker.sock:/var/run/docker.sock \
--name prtainer-test \
docker.io/portainer/portainer
  1. 登录查看本地镜像与容器
  • 访问http://IP:9000
  • 首次进入需要给管理员admin初始化密码
  • 单机版选择local即可
